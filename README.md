# Start-Hub: Blueprint Operativo 3X+

Este repositorio contiene la **Landing Page y Plataforma Interna** interactiva para la "Matriz de Aceleración 3X+" de Start-Hub. 

El sitio cumple un doble propósito:
1. **Herramienta de Capacitación Interna:** Un dashboard interactivo para consultar, buscar y filtrar rápidamente los 29 servicios del ecosistema Start-Hub.
2. **Landing Page Premium:** Una presentación corporativa de alto impacto para prospectos y clientes.

## Características Principales

- **Dashboard Interactivo:** Filtros en tiempo real por fase (EXplorar+, EXpandir+, EXcalar+), área y prioridad.
- **Acceso Restringido:** Capa de seguridad por contraseña en el frontend para evitar accesos no deseados de curiosos en la red. *(Clave requerida: `StartHubEstrategia.*`)*
- **Diseño Premium:** Estética dark-mode inspirada en interfaces de aceleradoras de élite, utilizando Glassmorphism y degradados fluidos.
- **100% Vanilla:** Construido completamente con HTML5, CSS3 y JavaScript ES6+, sin dependencias externas pesadas, garantizando carga rápida y facilidad de despliegue.
- **Recursos Integrados:** Sección para reproducir la Presentación Ejecutiva en video (MP4) y descargar el Documento Técnico del Blueprint (PDF).

## Estructura del Proyecto

- `index.html`: Estructura semántica principal de la landing page y el dashboard.
- `styles.css`: Estilos visuales centralizados, variables corporativas y animaciones.
- `app.js`: Lógica del filtro de datos, motor de búsqueda, ordenamiento, modales y validación de acceso.
- `Blueprint_Operativo__Matriz_3X+.mp4`: Video de presentación ejecutiva.
- `Blueprint_Operativo_3X .pdf`: Archivo descargable de manual técnico confidencial.
- `*.txt` y `*.pdf` adicionales: Archivos de consulta original de donde se alimentó la base de datos de servicios.

## Despliegue

Este proyecto está diseñado para funcionar perfectamente en **GitHub Pages**, Vercel, Netlify o cualquier servidor estático básico, ya que no requiere un backend de procesamiento activo.

1. Ve a la configuración (*Settings*) de tu repositorio en GitHub.
2. Navega a la sección **Pages**.
3. Selecciona la rama `main` y guarda.
4. Tu sitio web estará vivo y accesible al instante.
